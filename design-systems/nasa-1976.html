A typographic and visual system for editorial publishing — adapted from the modernist principles of the 1976 NASA Graphics Standards Manual. Built to be copied, pasted, and shipped.

§ 01.00
Color System

Color

Two accent hues against a neutral foundation. Red signals urgency and identity; blue signals interaction. The paper tone is intentionally warm — a printed-page reference, not a screen white.

Typography

§ 04.00
System in Use

Demo Post

A short article, set in the production system. This is what the entire blog should feel like in motion: typographic confidence, hard rules, restrained color.

Apr 26, 2026 · Filed: Design · 8 min read

How Two Designers Gave a Federal Agency a Voice

For seventeen years, the NASA Graphics Standards Manual quietly imposed a Swiss-modernist worldview on America's space program — and accidentally became the most influential design document of the late 20th century.

In 1974, the National Endowment for the Arts launched the Federal Design Improvement Program. The premise was simple: federal agencies were drowning in inconsistent, often ugly, communications. Hire actual designers and impose order.

NASA was an early participant. Richard Danne and Bruce Blackburn — partners at a New York firm — won the commission. What they delivered was not a logo refresh. It was a complete system of visual identity: typography, color, grid, photography, signage, vehicle livery, even the typography of the side of a rocket.

One Wordmark, Two Colors

FIG. 01 · Worm Logotype, 1976

The new wordmark — quickly nicknamed "the worm" — was unapologetically modern. The crossbars were removed from the A's. The letters flowed into each other. It looked, depending on who you asked, either like the future or like a typewriter ribbon trying to escape.

Source

The original 1976 manual is preserved in the Standards Manual reissue, available through Standards Manual NYC.

Why It Still Works

The system survives because it is, fundamentally, boring in the right way. It does not chase trends. It does not flex. It states facts in a clear voice and gets out of the way.

“Good design is as little design as possible.” Dieter Rams (often quoted by Danne)

Forty-six years later, the principles port cleanly into editorial design on the web. A condensed display face. A reliable body face. Two accent colors. A grid. Hard rules. That's the entire system. Everything else is restraint.

§ 05.00
Implementation

Drop-in CSS

Copy the token block below into your blog's stylesheet. Wire your existing components to the variables. Adjust the type scale to match your CMS, but keep the family pairings intact — they carry most of the identity.

/* ——— Tokens ——— */
:root {
  /* color */
  --nasa-red:     #FC3D21;
  --nasa-blue:    #0B3D91;
  --ink:          #0A0A0A;
  --paper:        #F1ECE0;
  --mute-1:       #6E6A60;
  --mute-2:       #B8B2A4;

  /* type */
  --font-display: "Archivo", sans-serif;
  --font-narrow:  "Archivo Narrow", sans-serif;
  --font-mono:    "JetBrains Mono", monospace;
}

/* ——— Base ——— */
body {
  font-family: var(--font-display);
  color: var(--ink);
  background: var(--paper);
}

h1, h2 {
  font-family: var(--font-narrow);
  font-weight: 800;
  text-transform: uppercase;
  letter-spacing: -0.02em;
}

a { color: var(--nasa-blue); }
a:hover { background: var(--nasa-blue); color: var(--paper); }

/* ——— Editorial accents ——— */
.accent { color: var(--nasa-red); }
hr { border: 0; border-top: 1px solid var(--ink); }
blockquote {
  border-block: 1px solid var(--ink);
  font-family: var(--font-narrow);
  text-transform: uppercase;
}
Do
  • — Reserve red for one element per screen
  • — Use uppercase for H1 and H2 only
  • — Set body in sentence case, generous leading
  • — Use hairline rules between sections
  • — Treat captions like instrument labels
Don't
  • — Tint red or blue with gradients
  • — Mix in a third accent color
  • — Use Inter, Roboto, or system defaults
  • — Round button corners
  • — Add drop shadows except as offset blocks
